# Support

Use the public docs first:

- [Product Guide](docs/help/product-guide.md)
- [Privacy Policy Draft](docs/legal/privacy-policy.md)
- [Security Policy](SECURITY.md)

Use GitHub issue forms for:

- reproducible bugs
- feature requests
- release-readiness feedback

Use private channels for:

- security reports
- sensitive customer data
- unreleased business details

Recommended operating model for launch:

- GitHub Issues for product feedback
- a lightweight triage ritual that labels `bug`, `feature`, `support`, and `launch-blocker`
- an automation step that mirrors labeled issues into an internal agent intake queue
