# Privacy Policy Draft

Last updated: April 1, 2026

This draft is for planning purposes only and is not legal advice. Review it with qualified counsel before publishing.

## Part 1: Summary

- Product: `flo`, a local-first desktop app for structuring and exporting AI-ready context
- Data collected by the app itself:
  - workspace files selected or created by the user
  - imported assets copied into the user's workspace folder
  - app preferences and recent workspace metadata stored locally
  - exported context copied to the clipboard or written to disk when the user chooses
- Data the app does not currently collect:
  - accounts
  - in-app analytics
  - cloud sync data
  - server-side workspace storage
- Primary jurisdictions to review before launch:
  - United States
  - California privacy law if California residents are targeted
  - GDPR if the product is offered to EU users

## Part 2: Full Privacy Policy

### 1. Scope

This Privacy Policy describes how `flo` handles information when you install and use the desktop application and related product pages.

### 2. Information We Collect

`flo` is designed to store work locally on your device. Depending on how you use the product, the application may process:

- workspace metadata such as project name, goal, and timestamps
- workspace content such as cards, connections, rich document text, comments, tags, and agent hints
- files you choose to import into a workspace
- local app preferences such as recent workspaces and interface settings
- exported context files and clipboard content when you explicitly trigger export or copy actions

`flo` does not currently require account registration and does not include built-in telemetry or hosted workspace storage.

### 3. How We Collect Information

We collect information only when you actively use the app:

- when you create, edit, save, or export a workspace
- when you import or attach files
- when you adjust local settings

If you later add a marketing website, waitlist, analytics, crash reporting, or payment flow, this policy must be updated before launch.

### 4. How We Use Information

We use information to:

- save and restore your workspace
- generate `context.md` exports
- preview workspace files
- create local history snapshots
- remember recent workspaces and settings on the device

### 5. Sharing

The current desktop app does not automatically send workspace content to company-controlled servers.

Information may leave your device only when you choose to:

- export files
- copy generated context to the clipboard
- open files with third-party applications
- share files or screenshots yourself

### 6. Retention

Retention is controlled primarily by the user because workspace data is stored locally.

- workspace files remain on disk until you delete them
- history snapshots remain in the workspace until removed by retention settings or manual deletion
- recent workspace metadata remains in local app storage until cleared or overwritten

### 7. Security

We use a local-first storage model, workspace folder boundaries, and Tauri capability controls to reduce unnecessary exposure. No software is perfectly secure, and users remain responsible for device security, local backups, and safe sharing behavior.
