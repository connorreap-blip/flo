A canvas-based knowledge architect where spatial thinking meets agent-native file hierarchy. Cards for humans. Markdown for agents.

The Problem

Your AI agent can't think
in hierarchies.

You dump an entire knowledge base into a prompt and hope for the best. Context windows overflow. Relationships disappear. Your agent sees a wall of text where you see a structured system.

Spatial tools are great for thinking but terrible for producing structured output. Document tools are great for depth but terrible for seeing the big picture.

flo bridges both. Sketch fast on the canvas. Attach documents only where depth is needed. Export a single structured context file that your agent actually understands. The visual map and the agent's context are the same thing.

How It Works

Three steps to structured thinking.

01

Map it spatially

Drop cards on an infinite canvas. Name them, type them (Project, Process, Reference, Brainstorm), connect them with typed arrows. Think fast. Structure emerges visually.

02

Add depth where needed

Any card can have a markdown document attached. A floating editor bubble opens inline. Write detail where it matters. Leave other cards as lightweight nodes. Not every thought needs a document.

03

Export for agents

One click generates context.md — a structured export with rules, hierarchy, workflows, scoped references, and documents. Your agent reads it directly. No API. No integration. Just files.

Card System

Four card types. Semantic meaning.

Cards are not generic sticky notes. Each type carries meaning that flows through to the file system, the hierarchy, and the agent export.

[PRJ]

Project

Top-level containers. Creates a folder in the file system. Requires a hierarchy document. The structural root of every initiative.

[PRC]

Process

Sequential workflows and procedures. Lives under a project. Connect with THEN edges to define order. The backbone of execution.

[REF]

Reference

Supporting material and cross-links. Accounts, policies, specs. Connected with scoped references that control how much context is surfaced.

[BRN]

Brainstorm

Ideas in progress. Automatically excluded from agent exports. Safe space for rough thinking that won't pollute your agent's context.

Context Scoping

Control what your agent sees.

Every reference edge has a scope. You decide the granularity. No other tool gives you this level of precision over agent context.

Title only
Just the card name in a "See also" list. Minimal context. Maximum brevity.
Summary
Card title + body text (what's visible on the card face). Enough for orientation without the full document.
Specific section
A targeted heading from the card's document. Surgical precision — pull exactly the section your agent needs.
Full card
Complete card including the full markdown document. Use sparingly — this is the maximum context injection.
